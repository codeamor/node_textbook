# 3.4 노드 내장 객체
## 1. global
> 노드의 전역 객체
- 브라우저의 window같은 역할
- 모든 파일에서 접근 가능
- window처럼 생략도 가능(console, require도 global 속성)

## 2. console 객체
- 객체를 로깅할 때엔 console.dir 이 좋음
- console.time() / console.timeEnd() => 두 코드 사이의 수행 시간
- console.trace: 호출 스택 로깅 (함수 안에서 사용)

## 3. __filename, __dirname
- __filename: 현재 파일 경로
- __dirname: 현재 폴더 경로

## 4. this
- anonymous에서 this는 module.exports 이고 함수 내에서는 global의 객체이다.

## 5. require 특성
- require가 맨 위에 올 필요 x
- require.cache에 한 번 require한 모듈에 대한 캐싱 정보가 들어 있음
  - 처음에는 하드에서 파일을 불러온다면 두 번째 부터는 메모리에서 불러옴
  - require.main은 노드 실행 시 첫 모듈을 가리킴
